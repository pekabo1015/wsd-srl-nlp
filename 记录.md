# 词义消歧与语义角色标注系统 - 项目记录

## 📌 应用总体总结

本项目是一个基于Python Streamlit框架的自然语言处理交互式Web分析系统，集成了两个核心功能模块。

**模块一（WSD-词义消歧）**：采用传统Lesk算法与深度学习BERT模型相结合的混合方法。用户输入包含多义词的句子，系统通过NLTK的Lesk算法识别WordNet词义，同时提取BERT上下文词向量（768维），并支持多句对比、计算余弦相似度来验证不同上下文中词义的语义相似性。

**模块二（SRL-语义角色标注）**：使用spaCy进行依存句法分析，通过精心设计的启发式规则自动识别谓词、施事者(A0)、受事者(A1)、地点修饰语(AM-LOC)和时间修饰语(AM-TMP)等语义角色。系统以结构化表格展示提取结果，并通过displacy库实时渲染依存关系图，帮助用户理解NLP处理过程。

两个模块共享统一的Streamlit UI框架，具有完整的错误处理、结果持久化和交互式展示能力，是深度学习与传统NLP方法的有机结合。

### 问题3：Streamlit session_state冲突错误
**提问时间**：2026-04-01

**错误现象**：
```
StreamlitAPIException: `st.session_state.sentence1` cannot be modified 
after the widget with key `sentence1` is instantiated.
```

**问题原因**：
Streamlit的widget（`st.text_area`、`st.text_input`）在指定`key`参数后，会自动管理对应的`session_state`。当按钮点击后直接修改与widget同名的`session_state`键时，会产生冲突。

**回答要点**：

#### 🔧 修复方案

使用不同的session_state键名存储分析结果，避免与widget的`key`参数冲突：

| 原键名 | 新键名 | 用途 |
|--------|--------|------|
| `sentence1` | `stored_sentence1` | 存储第一个句子 |
| `target_word` | `stored_target_word` | 存储目标多义词 |
| `lesk_result` | `stored_lesk_result` | 存储Lesk算法结果 |
| `embedding_result1` | `stored_embedding_result1` | 存储第一个句子的词向量 |

#### 🎯 SRL模块功能架构

##### 1️⃣ 用户交互
- **单个输入框**：接收要分析的句子
- **默认文本**：'Apple is manufacturing new smartphones in China this year.'
- **分析按钮**：触发SRL分析

##### 2️⃣ 轻量级近似实现（启发式规则）
由于完整的端到端SRL模型较重，使用spaCy进行依存句法分析，通过启发式规则近似提取语义角色。

**关键规则映射：**
- **谓词 (Predicate)**：句子中的主要动词（ROOT或VERB tag）
- **A0 (Agent 施事者)**：nsubj（名词性主语）
- **A1 (Patient 受事者)**：dobj（直接宾语）
- **AM-LOC (地点修饰语)**：介词短语表示地点
- **AM-TMP (时间修饰语)**：时间实体或时间修饰语

##### 3️⃣ 依存关系识别方法
```
使用spaCy的依存标记(dependency tags)：
├─ ROOT/VERB → 动词（谓词）
├─ nsubj → 主语（A0）
├─ dobj → 直接宾语（A1）
├─ prep → 介词（继续分析下游节点）
│  ├─ pobj (prep的对象)
│  │  ├─ in/at → AM-LOC
│  │  └─ in/during/on → AM-TMP
└─ advmod → 副词修饰（可能是时间/地点）
```

##### 4️⃣ 结构化展示（表格）
将提取的 [A0, 谓词, A1, AM-LOC, AM-TMP] 以Pandas DataFrame形式展示为表格：

| 语义角色 | 角色类型 | 提取文本 | 词性标签 |
|--------|--------|--------|--------|
| A0 (Agent) | 施事者 | Apple | NOUN |
| Predicate | 谓词 | manufacturing | VERB |
| A1 (Patient) | 受事者 | smartphones | NOUN |
| AM-LOC | 地点 | in China | PREP+NOUN |
| AM-TMP | 时间 | this year | DET+NOUN |

### 问题8：SRL模块session_state冲突
**提问时间**：2026-04-01

**错误现象**：
```
StreamlitAPIException: `st.session_state.srl_sentence` cannot be modified 
after the widget with key `srl_sentence` is instantiated.
```

**问题原因**：
SRL模块中，widget（`st.text_area`）使用`key="srl_sentence"`，后续代码直接修改同名的session_state，产生冲突。

**解决方案**：
使用`stored_`前缀或不同的键名存储分析结果。

| 原键名 | 新键名 | 用途 |
|--------|--------|------|
| `srl_sentence` | `stored_srl_sentence` | 存储输入的句子 |
| `srl_result` | `stored_srl_result` | 存储分析结果 |
| `srl_done` | `srl_analysis_done` | 标记分析完成状态 |

#### ✅ 修复完成
- ✅ 所有session_state键名更新
- ✅ 引用位置保持一致
- ✅ 冲突问题已解决

### 问题11：SRL模块依存关系图不显示
**提问时间**：2026-04-01

**问题现象**：
执行SRL分析后，表格正常显示，但依存关系图（displacy）没有加载显示。

**问题原因**：
1. `displacy.render()`使用`page=False`生成的HTML片段
2. `st.html()`可能不能完全正确渲染不含完整样式的HTML
3. displacy生成的样式需要完整的HTML框架

**解决方案**：

修改week5.py中SRL模块的displacy部分：

```python
# 旧代码
svg = displacy.render(doc, style="dep", manual=False, page=False)
st.html(svg)

# 新代码
html_dep = displacy.render(doc, style="dep", manual=False, page=True, options={"compact": False})
st.write(html_dep, unsafe_allow_html=True)
```

#### 📝 修改要点

1. ✅ **page参数**
   - 从`page=False`改为`page=True`
   - 生成包含完整CSS样式的HTML

2. ✅ **options参数**
   - 添加`options={"compact": False}`
   - 确保图表完整显示，不被压缩

3. ✅ **渲染方式**
   - 从`st.html()`改为`st.write(..., unsafe_allow_html=True)`
   - 更好的HTML兼容性

#### ✨ 修复效果
- ✅ displacy依存图正常显示
- ✅ 图表包含完整的CSS样式
- ✅ 依存关系清晰可见
- ✅ 用户可以验证SRL提取结果
